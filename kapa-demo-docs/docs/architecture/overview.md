- **Writer Agent**: Consumes the research report and style guidance, produces documentation file content.
- **Fix Assistant**: Applies user edits from the UI to fix content (used during review).

### Style Alignment and Guide Maintenance

The project uses a style guide to ensure consistency in documentation. This guide is derived from existing documentation practices and is maintained regularly to reflect any changes in standards. It is provided to both the Writer Agent and Fix Assistant, enabling them to generate and edit documentation that aligns with the established voice and formatting. Additionally, the style guide is generated by giving excerpts of the related documentation (which are selected by the researcher) and similar language and style is used in the writer. For more details on how the style guide is applied and updated, see the [Agents](./agents) documentation.

See [Agents](/docs/architecture/agents) for more detail.

### GitHub integration

The backend uses a **GitHub App** to read repository content, create branches and commits, open pull requests, and comment on issues. See [GitHub integration](/docs/api/github-integration).

## Data flow

```
Coverage Gap → Act → Issue → Research (→ Questions → Research) → Fix → Approve → PR
```

The frontend drives the flow by calling the backend API; Celery runs research and publish tasks. The backend is the single point of integration with GitHub and the only component that runs the agents.
